Lesson 15 (SCSS)

SCSS

Sass - это динамический язык для создания каскадных таблиц стиля (CSS). Также Sass называют CSS-препроцессором или препроцессорным языком. Он позволяет сделать разработку CSS более быстрой, простой и удобной.

Sass вводит переменные, операции, функции и миксины (примешивания).

Разработка на Sass удобна тем, что код становится более структурированным и гибким.

Чтобы использовать Sass(SCSS) код в проекте, его нужно преобразовать (скомпилировать) в CSS код.

Компилировать можно только с помощью специальных программ (Prepros App) или используя компиляторы, которые выполняются в командной строке.

Итак, код разрабатывается на SCSS, но затем файл компилируется в CSS файл и подключается в HTML.

В main.scss импортируем все остальные SCSS файлы:


@import 'components/normalize';

Порядок подключения очень важен: сначала переменные, потом миксины потом все остальное.

Нижнее подчеркивание и расширение файла указывать не нужно.

Нижнее подчеркивание в имени файла это инструкция компилятору, что этот файл не нужно компилировать отдельно.

Переменные

Переменные позволяют сохранить постоянно используемые значения в одном месте, а затем повторно использовать их в любом месте таблицы стилей, что облегчает внесение глобальных изменений до изменения одной строки кода.

В SCSS переменная объявляется с помощью символа $, при этом имя переменной и ее значение отделяются друг от друга двоеточием так, как это делается в CSS:


$main-color: #0982c1;
$site-width: 1024px;
$border-style: dotted;
body {
  color: $main-color;
  border: 1px $border-style $main-color;
  max-width: $site-width;
}

Математические операции

SCSS позволяет проводить вычисления прямо в CSS.

SCSS поддерживает следующие операторы +, -, *, / и %. При этом операторы + работает как сложение так и как соединение строк.


$max-width: 960px;
$base-font-size: 14px;
$heading-font-size: ($base-font-size + 10);
 
.heading h1{
  font-size: $heading-font-size;
}
.aside{
  width: (300px / $max-width * 100%);
}

Parent and root selector

Cимвол "&" вначале строки является ссылкой на родительский элемент

SCSS


li {
  float: left;
  & + & {
    pading-left: 20px;
  }
}
 
 
a {
  color: red;
  &:hover {
    color: white;
  }
}

CSS


li {
  float: left;
}
li + li {
  pading-left: 20px;
}
 
 
a {
  color: red;
}
a:hover {
  color: white;
}

Cимвол "&" в конце строки заменяет цепочку селекторов начиная с корнево элемента.

SCSS


.header {
  background: $black;
  .holder {
    padding: 20px 0;
      .home-page & {
        padding: 40px 0;
      }
    }
  }
}

CSS


.header {
  background: $black;
}
#header .holder {
  padding: 20px 0;
}
.home-page #header .holder{
  padding: 40px 0;
}

Операции с цветом

Встроенные функции для работы с цветом darken() и lighten() позволяют легко создавать hover-эффекты на свой вкус:


a{
  color: $base-link-color;
  &:hover, &:active, &:focus {
    color: lighten($base-link-color, 30%);
    text-decoration: none;
  }
}
textarea {
  margin: 0;
  padding: 4px 7px;
  border: 1px solid $base-input-border-color;
  &:focus {
    border-color: darken($base-input-border-color, 30%); 
  }
}

Наследование

Kоманда @extend позволяет наследовать свойства одного класса в другой.


.message {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}
 
.success {
  @extend .message;
  border-color: green;
}
 
.error {
  @extend .message;
  border-color: red;
}

Такой метод позволяет как использовать оригинальный класс в HTML так и создавать на его основе новые классы в CSS.

Служебные классы

Позволяют запоминать и повторно использовать целые наборы свойств. Служебные классы объявляются с помощью %:


// reset for list
%listreset {
  margin: 0;
  padding: 0;
  list-style: none;
}
 
#nav {
  @extend %listreset;
  float:left;
}

Служебные классы не переносятся в CSS:


#nav {
  margin: 0;
  padding: 0;
  list-style: none;
}
 
#nav {
  float: left;
}

Миксины (примешивания)

Миксины позволяют повторно использовать целые фрагменты SCSS кода и при необходимости даже передавать им аргументы. И что немаловажно, вы также можете указывать их значения по умолчанию.

Для объявления миксин используйте ключевое слово @mixin, а после него укажите выбранное вами имя миксина. Если есть необходимость в аргументах, то перечислите их в круглых скобках. Значения, устанавливаемые для аргумента по умолчанию, указываются при определении миксины через двоеточие.

Вызов (применение) миксины осуществляется ключевым словом @include после которого указывается имя миксины и опционально перечисляются в скобках необходимые параметры.

SCSS


@mixin make-box($bg: $white) {
  border-radius: 10px;
  padding: 15px;
  background: $bg;
  margin-bottom: $base-v-spacing;
}
.intro {
    @include make-box;
}
.news {
    @include make-box ($light-blue);
}

CSS


.intro {
  border-radius: 10px;
  padding: 15px;
  background: #fff;
  margin-bottom: 20px;
}
.news {
  border-radius: 10px;
  padding: 15px;
  background: #00c;
  margin-bottom: 20px;
}

Миксины не переносятся в CSS, т.о. миксины это более гибкий вариант служебных классов.

Создавайте миксины и служебные классы чтобы избавиться от повторения кода и вендорных префиксов. Не делайте миксины с кучей параметров. Они должны быть простыми и понятными. Всегда создавайте комментарий с описанием для своих миксин

Комментарии

SCSS придерживается привычного для CSS стиля комментариев:


/* Hello, I'm a css style comment */
.class{color: black;}

В SCSS также возможны однострочные комментарии, они считаются служебными, и не появляются в итоговом, скомпилированном выводе CSS:


// Hi, I'm a silent comment, I won't show up in your css
.class{color: white;}

Циклы

При умелом использовании циклы могут сильно сократить количество набираемого кода. Например при создании гридов:

SCSS


$columns: 12;
$gutter: 30px;
 
@for $i from 1 through $columns {
    .col-#{$i} {
        width: percentage(1 / $i);
        padding: 0 ($gutter / 2);
    }
}

CSS


.col-1 {
  width: 100%;
  padding: 0 15px;
}
 
.col-2 {
  width: 50%;
  padding: 0 15px;
}
 
.col-3 {
  width: 33.33333333%;
  padding: 0 15px;
}
 
...
 
.col-12 {
  width: 8.33333333%;
  padding: 0 15px;
}

Или вот такой перебор списка:

SCSS


@each $social-network in (facebook, youtube, flickr) {
  .#{$social-network} a {
    background-image: url('/images/ico-#{$social-network}.png');
    ...
  }
}

CSS


.facebook a {
  background-image: url("/images/ico-facebook.png");
  ...
}
 
.youtube a {
  background-image: url("/images/ico-youtube.png");
  ...
}
 
.flickr a {
  background-image: url("/images/ico-flickr.png");
  ...
}

SCSS + Responsive

SCSS позволяет вкладывать медиа-запросы внутрь CSS-правил, т.о. свойства элемента для всех девайсов записываются в одном месте. Для написания медиа-развилок мы будем использовать миксину @media().

Пример записи свойств для мобильных устройств:

SCSS


.sidebar {
  width: 300px;
  float:left;
  @include media('<tablet') {
    width: 100%;
    float:none;
  }
}

CSS


.sidebar {
  width: 300px;
  float:left;
}
@media ( max-width:767px ) {
  .sidebar {
    width: 100%;
    float:none;
  }
}

Компилятор SCSS также легко обрабатывает вложенные медиа-запросы:

SCSS


.icon {
    background: url(../images/icon.png) no-repeat;
 
    @include breakpoint('<tablet') {
        background: url(../images/icon-sm.png) no-repeat;
 
        @include breakpoint('retina2x') {
            background: url(../images/icon-sm-2x.png) no-repeat;
        }
    }
}

CSS


.icon {
  background: url(../images/icon.png) no-repeat;
}
@media (max-width: 767px) {
  .icon {
    background: url(../images/icon-sm.png) no-repeat;
  }
}
@media (max-width: 767px) and (min-resolution: 144dpi), 
       (max-width: 767px) and (-webkit-min-device-pixel-ratio: 1.5) {
  .icon {
    background: url(../images/icon-sm-2x.png) no-repeat;
  }
}

Media breakpoints


$breakpoints: (
	'xs-phone': 320px,
	'phone': 576px,
	'tablet': 768px,
	'desktop': 992px,
	'widescreen': 1200px
);

// keywords
$media-expressions: (
	'screen': 'screen',
	'print': 'print',
	'handheld': 'handheld',
	'landscape': '(orientation: landscape)',
	'portrait': '(orientation: portrait)',
	'retina2x': '(-webkit-min-device-pixel-ratio: 1.5), (min-resolution: 144dpi)',
	'retina3x': '(-webkit-min-device-pixel-ratio: 2.5), (min-resolution: 240dpi)'
);

// ligatured operators ≥ ≤
